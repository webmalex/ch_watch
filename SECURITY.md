# Безопасность

## Поддерживаемые версии

Исправления безопасности попадают только в актуальную ветку `master`. После начала формальных релизов исправления также будут porter'иться в последний tagged release.

## Поверхность атаки

`ch_watch` — локальный CLI инструмент без сетевых сервисов. Серверная поверхность атаки минимальна. Тем не менее, уязвимости возможны в:

- **Зависимостях** Go (проверяются через `govulncheck`).
- **Путях выполнения** — инструмент читает файлы с диска и передаёт их в `clickhouse client`/`clickhouse local` через `stdin`. Path traversal или непредсказуемое поведение при обработке имён файлов могут создать риски.
- **Выполнении внешних команд** — `ch_watch` запускает ClickHouse binary; проблемы в конфигурации окружения или подмена binary по `PATH` влияют на безопасность.

## Сообщить об уязвимости

Не открывайте public issues для отчётов об уязвимостях до завершения координации раскрытия. Используйте приватный канал:

https://github.com/webmalex/ch_watch/security/advisories/new

## Что включить в отчёт

- Описание уязвимости и шаги для воспроизведения.
- Затронутые версии или коммиты.
- Предлагаемое исправление, если есть.
- Любую дополнительную информацию, которая поможет быстрее разобраться.

## Ожидания

Ответ в течение 72 часов, по возможности. Проект поддерживается на best-effort основе — временные рамки зависят от сложности и загруженности. После подтверждения и координации fix будет включён в ближайший коммит `master` и, при наличии релизов, в соответствующий release patch.
